## Try Danube

Download the latest binary from the [releases page](https://github.com/danube-messaging/danube/releases) or pull the [container image](https://github.com/danube-messaging/danube/pkgs/container/danube-broker).

<Tabs>
	<TabItem label="Standalone">
		Start a broker and test with the CLI:

		```bash
		# Start the broker
		danube-broker --mode standalone --data-dir ./danube-data

		# Produce messages
		danube-cli produce -s http://127.0.0.1:6650 -t /default/demo -c 10 -m "Hello, Danube!"

		# Consume messages
		danube-cli consume -s http://127.0.0.1:6650 -t /default/demo -m my_sub
		```

		Or run a single broker in a container:

		```bash
		docker run -p 6650:6650 -p 50051:50051 \
		  ghcr.io/danube-messaging/danube-broker:latest \
		  --mode standalone --data-dir /data
		```
	</TabItem>
	<TabItem label="Cluster">
		Run the following command on each of your 3 nodes to form a Raft consensus cluster:

		```bash
		danube-broker --config-file danube_broker.yml \
		  --broker-addr 0.0.0.0:6650 --raft-addr 0.0.0.0:7650 \
		  --data-dir ./data/raft \
		  --seed-nodes "node1:7650,node2:7650,node3:7650"
		```

		Or use Docker Compose to start a 3-node cluster with Prometheus monitoring:

		```bash
		git clone https://github.com/danube-messaging/danube.git
		cd danube/docker/quickstart
		docker-compose up -d
		```

		Check cluster health:

		```bash
		danube-admin brokers list
		danube-admin brokers balance
		```
	</TabItem>
	<TabItem label="Edge (IoT)">
		MQTT gateway for IoT devices at the edge. Validates payloads against schemas, buffers into a local WAL that survives outages, and replicates to your central cluster over gRPC.

		```bash
		danube-broker --mode edge --data-dir ./edge-data --edge-config edge.yaml
		```
	</TabItem>
</Tabs>

## Why Danube?

<CardGrid stagger>
	<Card title="One Binary" icon="setting">
		Run as **Standalone** for development, **Cluster** for production, or **Edge** for IoT ingestion, all from the same `danube-broker` binary with no external dependencies.
	</Card>
	<Card title="Pub/Sub & Streaming" icon="rocket">
		Publish to topics, consume via named subscriptions. Use best-effort dispatch for fire-and-forget real-time workloads, or persistent dispatch with WAL-backed at-least-once delivery, acknowledgments, and dead-letter queues.
	</Card>
	<Card title="Resilient Clusters" icon="approve-check">
		Embedded Raft consensus with automatic leader election, failover and topic reconciliation on restart. Intelligent load manager monitors broker health and rebalances topics to prevent hotspots. Scale horizontally with zero downtime.
	</Card>
	<Card title="Cloud-Native Storage" icon="open-book">
		Local WAL for fast writes with background export to S3, GCS, or Azure Blob via OpenDAL. Tiered reads serve from cache when possible, falling back to durable segments. Topics move between brokers without data loss.
	</Card>
	<Card title="Schema Registry" icon="document">
		Built-in schema management with JSON Schema, Avro, and Protobuf. Version control with backward, forward, and full compatibility enforcement, no separate registry service to deploy.
	</Card>
	<Card title="AI-Native Admin" icon="star">
		Manage your cluster with natural language via the Model Context Protocol. 40+ tools and guided workflows for health checks, schema evolution, and rebalancing through Claude, Cursor, or any MCP-compatible assistant.
	</Card>
</CardGrid>
